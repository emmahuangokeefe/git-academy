## How can I use git for my work? * Text files are best (like html,css) * Binary files can work to (like docx,odt,jpg,gif) * Pretty much anything you give it * Git tracks an entire directory of files * Runs on Linux, Mac OSX and Windows
## How does it work? * This is all kept in the .git folder * You store your 'saves' in a commit * Sharing work is done through URLs
* Git tracks directories * using .git folder * commits = saves with comments * sharing = urls
## Setup your git Open a shell, and... ```bash # Install git sudo apt-get install git # Configure git, stores these in ~/.gitconfig git config --global user.name "my full name" git config --global user.email "your_email@example.com" git config --global core.editor vim git config --global color.ui auto ```

Your first commands

  • git init <-- Create a new repository
  • git add . <-- Get ready to save my work
  • git status <-- Tell me what's going on
  • git commit <-- Save 'added' work in a commit
  • git log <-- Show changes that lead up to my current state
## Create your repo ```bash # Create a new homework repository mkdir homework cd homework git init git status ``` Should show something like...
```bash mbgray@monolyth ~ % mkdir homework mbgray@monolyth ~ % cd homework mbgray@monolyth ~/homework % git init Initialized empty Git repository in /home/mbgray/homework/.git/ mbgray@monolyth ~/homework (git)-[master] % git status On branch master Initial commit nothing to commit (create/copy files and use "git add" to track) ```
## Create some files ``` vim lab1.txt vim lab2.txt git status ``` Should show something like...
```bash mbgray@monolyth ~/homework (git)-[master] % vim lab1.txt mbgray@monolyth ~/homework (git)-[master] % vim lab2.txt mbgray@monolyth ~/homework (git)-[master] % git status On branch master Initial commit Untracked files: (use "git add ..." to include in what will be committed) lab1.txt lab2.txt nothing added to commit but untracked files present (use "git add" to track) ```
## Add files ready to commit ```bash git add . git status ``` Should show something like...
```bash mbgray@monolyth ~/homework (git)-[master] % git add . mbgray@monolyth ~/homework (git)-[master] % git status On branch master Initial commit Changes to be committed: (use "git rm --cached ..." to unstage) new file: lab1.txt new file: lab2.txt ```
## Commit your work ```bash git commit ``` Should put you into a text editor requesting a commit message ``` # Please enter the commit message for your changes. Lines starting # with '#' will be ignored, and an empty message aborts the commit. # On branch master # # Initial commit # # Changes to be committed: #»······new file: lab1.txt #»······new file: lab2.txt # ``` Type something meaningful, like "Finished lab 1 + lab 2" When you're ready, save and quit with :wq
## Check out the log ```bash git log ``` Should show something like... ```bash mbgray@monolyth ~/homework (git)-[master] % git log commit 93b4e770e61aaf1b0b7b20cb2b69c4a99278a664 Author: Matthew B. Gray Date: Thu Jan 4 01:30:52 2015 +1300 Finished lab 1 + lab 2 ``` Note, because this is small, it wont open in a pager

## Edit your files ```bash git diff vim lab1.txt # Check what changed git status git diff ``` Should show something like...
```bash mbgray@monolyth ~/homework (git)-[master] % git diff mbgray@monolyth ~/homework (git)-[master] % vim lab1.txt mbgray@monolyth ~/homework (git)-[master] % git status On branch master Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git checkout -- ..." to discard changes in working directory) modified: lab1.txt no changes added to commit (use "git add" and/or "git commit -a") mbgray@monolyth ~/homework (git)-[master] % git diff diff --git a/lab1.txt b/lab1.txt index b1a17ba..f55c2ce 100644 --- a/lab1.txt +++ b/lab1.txt @@ -1 +1,2 @@ old stuff +new stuff old stuff -removed stuff old stuff ```
## Throw away your changes ```bash git status git checkout . git status ``` Should give something like...
```bash mbgray@monolyth ~/homework (git)-[master] % git status On branch master Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git checkout -- ..." to discard changes in working directory) modified: lab1.txt no changes added to commit (use "git add" and/or "git commit -a") mbgray@monolyth ~/homework (git)-[master] % git checkout . mbgray@monolyth ~/homework (git)-[master] % git status On branch master nothing to commit, working directory clean ```
